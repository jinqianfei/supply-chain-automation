### 3.2 Phase 1: 数据库迁移

#### 目标云数据库选型

| 方案 | 提供商 | 免费额度 | 推荐度 |
|------|--------|---------|--------|
| **Neon** | Neon Technologies | 0.5GB存储/无限分支 | ⭐⭐⭐⭐⭐ |
| Supabase | Supabase | 500MB存储 | ⭐⭐⭐⭐ |
| Railway | Railway | $5免费额度 | ⭐⭐⭐ |
| 云VM自建 | 阿里云/腾讯云 | 按量付费 | ⭐⭐⭐ |

**推荐 Neon**:
- 原生 PostgreSQL，兼容性好
- 免费额度足够中小规模使用
- 支持分支（可用于测试环境）

#### 操作步骤

**Step 1: 导出本地数据**
```bash
# 本地执行（确保本地neo数据库可访问）
pg_dump -U jinqianfei -d neo -h localhost -p 5432 -f neo_backup.sql

# 或导出特定表
pg_dump -U jinqianfei -d neo -h localhost -p 5432 \
  -t store_list \
  -t shipper_sku_mapping \
  -t warehouse_code_mapping \
  -t customer \
  -f neo_tables.sql
```

**Step 2: 注册 Neon 并创建项目**
```
1. 访问 https://neon.tech
2. 注册并登录
3. 创建新项目:
   - Name: ai-order
   - Region: 华北/华东（如使用阿里云，推荐日本或新加坡）
   - PostgreSQL Version: 15
4. 创建同名数据库: neo
```

**Step 3: 导入数据到 Neon**
```bash
# 安装 psql 客户端（如果没有）
# macOS: brew install postgresql@15

# 连接 Neon 并导入
psql "postgresql://username:password@ep-xxx-xxx-123456.us-east-2.aws.neon.tech/neon?sslmode=require"

# 在 psql shell 中执行
\i neo_backup.sql

# 或导入表数据
\i neo_tables.sql
```

**Step 4: 更新数据库配置**
```python
# 云端 db_config
db_config = {
    "host": "ep-xxx-xxx-123456.us-east-2.aws.neon.tech",  # Neon 提供的 Host
    "port": 5432,
    "database": "neo",
    "user": "your_neon_username",
    "password": "your_neon_password",  # 建议用环境变量
    "sslmode": "require"
}
```

### 3.3 Phase 2: 云服务器部署 OpenClaw

#### 服务器要求

| 配置项 | 最低要求 | 推荐配置 |
|--------|---------|---------|
| CPU | 2核 | 2核 |
| 内存 | 4GB | 4GB |
| 系统 | Ubuntu 22.04 | Ubuntu 22.04 LTS |
| 磁盘 | 40GB SSD | 60GB SSD |
| 网络 | 公网IP | 固定公网IP |

**推荐云服务商**:
- 阿里云 ECS (轻量应用服务器)
- 腾讯云 Lighthouse
- AWS Lightsail

#### 安装步骤

**Step 1: 连接服务器**
```bash
ssh root@your-server-ip
```

**Step 2: 安装 Node.js 18+**
```bash
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs
npm install -g openclaw
```

**Step 3: 创建工作目录**
```bash
mkdir -p /opt/openclaw
cd /opt/openclaw

# 创建目录结构
mkdir -p config skills data/orders data/output logs
```

**Step 4: 配置 OpenClaw**
```bash
# 创建主配置文件
cat > /opt/openclaw/config/openclaw.yaml << 'EOF'
workspace: /opt/openclaw
host: 0.0.0.0
port: 3000

# 飞书配置
feishu:
  enabled: true
  app_id: ${FEISHU_APP_ID}
  app_secret: ${FEISHU_APP_SECRET}
  verification_token: ${FEISHU_VERIFICATION_TOKEN}
  encrypt_key: ${FEISHU_ENCRYPT_KEY}

# Agents 配置
agents:
  default:
    model: minimax-portal/MiniMax-M2.7
    timeout: 300

# Skills 路径
skills:
  path: /opt/openclaw/skills
EOF
```

**Step 5: 创建 .env 文件**
```bash
cat > /opt/openclaw/.env << 'EOF'
# 数据库配置
DATABASE_URL=postgres://user:password@host:5432/neo?sslmode=require

# LLM API
MINIMAX_API_KEY=your_minimax_api_key

# 飞书配置
FEISHU_APP_ID=cli_xxxxxxxxxxxxxx
FEISHU_APP_SECRET=xxxxxxxxxxxxxxxxxxxxxxxx
FEISHU_VERIFICATION_TOKEN=xxxxxxxxxxxxxxxx
FEISHU_ENCRYPT_KEY=

# OpenClaw
OPENCLAW_PORT=3000
OPENCLAW_WORKSPACE=/opt/openclaw
EOF
```

**Step 6: 配置 systemd 服务**
```bash
sudo nano /etc/systemd/system/openclaw.service
```

写入内容：
```ini
[Unit]
Description=OpenClaw AI Agent
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/openclaw
ExecStart=/usr/bin/openclaw gateway start
Restart=always
RestartSec=10
Environment=NODE_ENV=production
EnvironmentFile=/opt/openclaw/.env

[Install]
WantedBy=multi-user.target
```

启动服务：
```bash
sudo systemctl daemon-reload
sudo systemctl enable openclaw
sudo systemctl start openclaw
sudo systemctl status openclaw
```

**Step 7: 配置防火墙**
```bash
# 开放 3000 端口（OpenClaw）
sudo ufw allow 3000/tcp

# 或使用云服务器安全组
# 在云控制台添加规则：入方向 TCP 3000
```

#### 需修改的文件

**1. `__init__.py` - 数据库连接配置**

找到并修改 `_load_warehouse_mapping()` 和 `__init__()` 中的数据库配置：

```python
# 修改前（本地）
db_config = {
    "host": "localhost",
    "port": 5432,
    "database": "neo",
    "user": "jinqianfei"
}

# 修改后（云端 - 从环境变量读取）
import os
db_config = {
    "host": os.environ.get("DB_HOST", "localhost"),
    "port": int(os.environ.get("DB_PORT", 5432)),
    "database": os.environ.get("DB_NAME", "neo"),
    "user": os.environ.get("DB_USER", "jinqianfei"),
    "password": os.environ.get("DB_PASSWORD", ""),
    "sslmode": os.environ.get("DB_SSLMODE", "require")
}
```

### 3.5 Phase 4: 飞书集成配置

#### 飞书应用配置

**Step 1: 修改飞书应用回调地址**
```
1. 登录飞书开放平台: https://open.feishu.cn/app
2. 选择你的应用 (AI建单助手)
3. 进入「添加应用能力」→「机器人」
4. 配置请求地址:
   开发环境: https://your-server-ip:3000/webhook/feishu
   生产环境: https://your-domain.com/webhook/feishu
```

**Step 2: 配置公网访问**

方案A: **固定公网IP** (推荐)
```
1. 云服务器绑定固定公网IP
2. 安全组规则:
   - 入方向: TCP 3000 端口
   - 来源: 0.0.0.0/0 或飞书IP段
3. 测试访问: curl https://your-public-ip:3000/health
```

方案B: **Nginx 反向代理 + 域名**
```nginx
# /etc/nginx/sites-available/openclaw
server {
    listen 443 ssl;
    server_name your-domain.com;

    ssl_certificate /etc/ssl/certs/openclaw.crt;
    ssl_certificate_key /etc/ssl/private/openclaw.key;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

## 四、数据备份策略

### 4.1 自动备份脚本
```bash
#!/bin/bash
# /opt/openclaw/backup.sh

DATE=$(date +%Y%m%d)
BACKUP_DIR=/opt/openclaw/backups

# 备份数据库
pg_dump "postgresql://user:pass@host:5432/neo?sslmode=require" \
  -f "$BACKUP_DIR/neo_backup_$DATE.sql"

# 保留最近30天备份
find $BACKUP_DIR -name "neo_backup_*.sql" -mtime +30 -delete

# 上传到 OSS (可选)
# ossutil cp $BACKUP_DIR/neo_backup_$DATE.sql oss://your-bucket/backups/
```

### 4.2 Cron 定时任务
```bash
# 每天凌晨2点执行备份
0 2 * * * /opt/openclaw/backup.sh >> /opt/openclaw/logs/backup.log 2>&1
```

## 七、应急预案

### 7.1 回滚方案
- 保留本地开发环境作为备份
- 切换回本地只需修改数据库连接配置

### 7.2 故障排除

| 故障现象 | 可能原因 | 解决方案 |
|---------|---------|---------|
| 无法连接数据库 | SSL配置/网络 | 检查 `sslmode=require` |
| 飞书消息无响应 | Webhook URL错误 | 确认公网可达 |
| OpenClaw启动失败 | 端口被占用 | `lsof -i:3000` |
| Skill加载失败 | 文件路径错误 | 检查skills目录配置 |
