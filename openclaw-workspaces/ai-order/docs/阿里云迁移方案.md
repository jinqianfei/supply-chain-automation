## 二、迁移策略

### 方案选择

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| **A. 数据库保留 RDS，只迁移计算** | 零数据迁移，低风险 | 跨云延迟（阿里云→AWS RDS） | ⭐⭐⭐ |
| **B. 全量迁移（含数据库）** | 全部在阿里云，低延迟 | 需迁移 RDS 数据，停机时间更长 | ⭐⭐ |
| **C. 数据库用阿里云 RDS + pg_dump 迁移** | 全阿里云生态，延迟最低 | 最复杂，需配置新 RDS | ⭐⭐⭐⭐ (长期最优) |

**建议**：先用 **方案 A** 快速上线（数据库继续用 AWS RDS），后续按需升级到方案 C。

## 三、阿里云 ECS 规格建议

| 配置项 | 推荐值 | 说明 |
|--------|--------|------|
| **实例规格** | ecs.c6.xlarge (4vCPU 8GB) 或 ecs.c7.xlarge | 当前 EC2 负载不高，够用 |
| **系统盘** | 40GB SSD (ESSD Entry) | 当前 `.openclaw/` 就 240MB |
| **数据盘** | 不需要（数据在 RDS） | 除非方案 B/C |
| **操作系统** | Alibaba Cloud Linux 3 (兼容 CentOS) | 或 Ubuntu 22.04 |
| **地域** | 华东1（杭州）/ 华东2（上海） | 国内访问最优 |
| **带宽** | 按量 5Mbps 或 固定 3Mbps | Cloudflare 隧道不需要大带宽 |
| **安全组** | 开放 22 (SSH), 18789 (OpenClaw 内部) | Cloudflare 隧道不需要开放公网端口 |

**预估月费用**：约 ¥300-500/月（按量）或 ¥200-300/月（包年）

## 四、迁移步骤（方案 A：数据库保留 RDS）

### Phase 1：准备阿里云 ECS（约 30 分钟）

```bash
# 1. 购买 ECS 实例（控制台操作）
#    - 规格：ecs.c6.xlarge
#    - 系统：Alibaba Cloud Linux 3
#    - 安全组：开放 22 端口

# 2. SSH 登录新服务器
ssh root@<阿里云ECS_IP>

# 3. 创建普通用户
adduser aiorder
usermod -aG wheel aiorder
su - aiorder

# 4. 安装 NVM + Node.js
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
source ~/.bashrc
nvm install 22
node --version  # v22.x.x

# 5. 安装 Python 3.9+
sudo yum install -y python3 python3-pip  # Alibaba Cloud Linux
# 或 sudo apt install -y python3 python3-pip  # Ubuntu
python3 --version

# 6. 安装 psycopg2 (PostgreSQL 驱动)
pip3 install psycopg2-binary

# 7. 安装 OpenClaw
npm install -g openclaw
openclaw --version

# 8. 安装 Cloudflared
curl -L https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64 -o /usr/local/bin/cloudflared
chmod +x /usr/local/bin/cloudflared
```

### Phase 2：迁移工作区数据（约 10 分钟）

```bash
# 在 AWS EC2 上打包
ssh ec2-user@13.212.17.85
cd /home/ec2-user
tar czf /tmp/ai-order-backup.tar.gz \
  ai-order/ \
  .openclaw/openclaw.json \
  .openclaw/openclaw.json.last-good \
  .openclaw/agents/ \
  .openclaw/logs/

# 传输到阿里云 ECS（从本地中转或直接 scp）
scp -i ~/.ssh/openclaw-ec2.pem ec2-user@13.212.17.85:/tmp/ai-order-backup.tar.gz /tmp/
scp /tmp/ai-order-backup.tar.gz aiorder@<阿里云ECS_IP>:/tmp/

# 在阿里云 ECS 上解压
ssh aiorder@<阿里云ECS_IP>
cd /home/aiorder
tar xzf /tmp/ai-order-backup.tar.gz
```

### Phase 3：配置环境（约 15 分钟）

```bash
# 1. 创建 .env 文件
cat > /home/aiorder/ai-order/.env << 'EOF'
DB_HOST=agenthub-db.cjys0msc4x8s.ap-southeast-1.rds.amazonaws.com
DB_PORT=5432
DB_NAME=neo
DB_USER=agenthub
DB_PASSWORD=Agenthub_RDS_2026!c9fce7060773
REDIS_HOST=localhost
REDIS_PORT=6380
REDIS_PASSWORD=openclaw123
MINIMAX_API_KEY=<从AWS的.env复制>
EOF

# 2. 创建启动脚本
cat > /home/aiorder/start-openclaw.sh << 'EOF'
#!/bin/bash
source /home/aiorder/ai-order/.env
export DB_HOST DB_PORT DB_NAME DB_USER DB_PASSWORD
export MINIMAX_API_KEY
nohup node /usr/lib/node_modules/openclaw/dist/index.js gateway --port 18789 > /home/aiorder/.openclaw/logs/gateway.log 2>&1 &
echo "OpenClaw started with PID $!"
EOF
chmod +x /home/aiorder/start-openclaw.sh

# 3. 配置 OpenClaw
# 复制 openclaw.json 配置（修改密码、域名等）
cp /home/aiorder/.openclaw/openclaw.json.last-good /home/aiorder/.openclaw/openclaw.json
# 编辑 openclaw.json，更新：
# - gateway.auth.password (如需修改)
# - gateway.auth.allowedOrigins (新的 Cloudflare URL)

# 4. RDS 安全组：添加阿里云 ECS IP 的入站规则
# 在 AWS RDS 控制台 → 安全组 sg-085c8ff5ca5ed1e9e → 添加入站规则：
#   类型: PostgreSQL (5432)
#   来源: <阿里云ECS公网IP>/32
```

### Phase 4：启动服务（约 10 分钟）

```bash
# 1. 安装 Redis（如果需要）
sudo yum install -y redis  # 或 apt install redis
sudo systemctl start redis
sudo systemctl enable redis
# 修改 Redis 端口为 6380 + 设置密码
sudo sed -i 's/port 6379/port 6380/' /etc/redis/redis.conf
echo "requirepass openclaw123" | sudo tee -a /etc/redis/redis.conf
sudo systemctl restart redis

# 2. 启动 OpenClaw
cd /home/aiorder
bash start-openclaw.sh
sleep 3
# 验证
curl -s http://localhost:18789/ | head -5

# 3. 启动 Cloudflare Tunnel
nohup cloudflared tunnel --url http://localhost:18789 > /tmp/cloudflared.log 2>&1 &
sleep 5
grep -o 'https://[a-z0-9-]*.trycloudflare.com' /tmp/cloudflared.log | head -1
# 记录新的 Tunnel URL

# 4. 更新 openclaw.json 的 allowedOrigins
# 将新的 Cloudflare URL 加入配置
```

### Phase 5：验证（约 15 分钟）

## 五、关键风险 & 注意事项

### 5.1 RDS 跨云延迟

| 场景 | 预估延迟 |
|------|---------|
| AWS 新加坡 → 阿里云新加坡 | ~5ms (同地域) |
| AWS 新加坡 → 阿里云杭州 | ~50-80ms |
| AWS 新加坡 → 阿里云上海 | ~50-80ms |

**影响**：单次订单处理约 5-10 次 DB 查询 → 额外增加 250-800ms
**建议**：如果延迟不可接受，选阿里云新加坡地域，或后续迁移数据库到阿里云 RDS

### 5.2 Cloudflare Tunnel

- 当前用的是**临时 Tunnel**（每次重启 URL 变化）
- 迁移后需要重新建立 Tunnel
- **建议**：配置固定域名 Tunnel（需 Cloudflare 账号 + DNS 配置）

### 5.3 飞书 Channel 配置

- 当前 EC2 上**未配置飞书 Channel**（金姐通过本地 Mac 的 OpenClaw 连飞书）
- 迁移后如果要用阿里云 ECS 的 OpenClaw 连飞书，需要配置飞书 App

### 5.4 API Keys

- MiniMax API Key：需要从 AWS `.env` 复制
- 其他 API Key（如 OpenAI、Qwen）：如有需要一并配置

## 六、回滚方案

如果阿里云部署有问题，随时可以切回 AWS EC2：

```bash
# AWS EC2 上的服务不要停
# 只需重新建立 Cloudflare Tunnel 即可恢复
ssh ec2-user@13.212.17.85
nohup cloudflared tunnel --url http://localhost:18789 > /tmp/cloudflared.log 2>&1 &
```

## 七、长期优化建议（方案 C）

如果确认长期使用阿里云，建议后续做：

1. **数据库迁移到阿里云 RDS PostgreSQL**
   - `pg_dump` 导出 → `pg_restore` 导入
   - 停机时间 < 10 分钟
   - 消除跨云延迟

2. **固定域名 + SSL**
   - 阿里云 DNS 解析 + Let's Encrypt 证书
   - 或 Cloudflare 固定 Tunnel

3. **systemd 服务**
   - 把 OpenClaw 注册为 systemd 服务
   - 开机自启 + 自动重启 + 日志管理

4. **监控告警**
   - 阿里云云监控 + 自定义告警
   - 进程存活检查 + 数据库连接检查
