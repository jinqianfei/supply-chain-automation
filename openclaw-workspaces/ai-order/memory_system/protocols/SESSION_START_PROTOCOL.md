# 新会话启动协议

**触发时机：** 每次开始一个新的会话时（主 session 或 sub-agent）。

> **原则：按需读取，不强制预加载。**
> 只需要知道"有什么可查"，不需要在启动时就读完所有记忆。

---

## 启动问询（简化版）

直接问用户今天要做什么，不需要主动汇报。

```
你好！今天要做什么？
```

---

## 按需读取触发条件

| 用户说/提到 | 我去读什么 |
|------------|------------|
| "继续上次XXX"、"上次那个项目" | `projects/<项目>/sessions/` 最新会话 + `PROJECT.md` |
| "继续上次XXX"、"还有事没完" | 读对应项目的 `problems/PENDING.md`，检查紧急事项 |
| "我记得之前说过..."、"之前那个..." | `MEMORY.md` 搜索相关段落 |
| "密码"、"账号"、"密钥" | `memory/credentials/INDEX.md` |
| "XX项目的背景/状态/问题" | `projects/<项目>/PROJECT.md` + `problems/INDEX.md` |
| "今天的待办"、"还有哪些没做完" | 各项目 `problems/PENDING.md`紧急项 + `MEMORY.md` 待办 |
| "之前那个紧急事项" | 读 `PENDING.md` 找对应 ID |
| "Skill版本"、"谁改了什么" | 对应 Skill 的 `CHANGELOG.md` |

---

## 主动提醒规则

当用户提到某项目时，我应主动检查该项目的 `PENDING.md`：
- 有🔴 紧急项 → 告知用户："还有 X 件紧急事项待处理"
- 有 🟡 进行中项 → 询问是否继续
- 有超过3 天无更新的 → 询问是否继续或放弃

---

## 禁止行为

- ❌ 启动时不读 MEMORY.md 就假装忘了之前说过的事
- ❌ 被问到"之前那个项目"时说不记得
- ❌ 被问到凭证时说没记住（应该立即查 INDEX.md）
- ❌ 不确定项目状态时说"忘了"，应该去读 PROJECT.md
- ❌ 用户问"还有什么事"时说不知道（应该查 PENDING.md）
